# Contribution Guidelines for Eaglercraft-1.21.11

Welcome to the Eaglercraft-1.21.11 project! We appreciate your interest in contributing. This document outlines our guidelines for contributing to this project, including how to report bugs, suggest enhancements, and submit your changes.

## Reporting Bugs
- **Step 1:** Search the issues to see if your bug has already been reported.
- **Step 2:** If it hasn't, create a new issue with the label 'bug'. Include detailed description, steps to reproduce, expected behavior, and any relevant screenshots.

## Suggesting Enhancements
- **Step 1:** Check if your enhancement proposal has already been suggested by searching the issues.
- **Step 2:** If it hasn't, open a new issue with the label 'enhancement'. Provide a detailed description of your suggestion and its benefits.

## Pull Request Process
1. **Fork the repository:** Click on the 'Fork' button at the top right.
2. **Clone your fork:** This copies your fork to your local machine.
   ```
git clone https://github.com/your-username/Eaglercraft-1.21.11.git
```
3. **Create a new branch:**
   ```
git checkout -b your-branch-name
```
4. **Make your changes** and commit them with a descriptive commit message.
5. **Push your branch:**
   ```
git push origin your-branch-name
```
6. **Create a new pull request** from your branch to the main branch of the original repository.

## Code Style Guidelines
- Follow the existing coding style in the codebase, which is predominantly set up for readability and consistency.
- Include comments to explain complex logic.
- Ensure that code formatting adheres to standard conventions.

## Development Setup
1. **Clone the repository:**
   ```
git clone https://github.com/dinoragalindofuentes1-crypto/Eaglercraft-1.21.11.git
```
2. **Install dependencies:** Follow the instructions in the `README.md` to set up the development environment and install any necessary dependencies.
3. **Run the application:** Ensure that the application runs correctly locally before making any contributions.

## Commit Messages
- Use the present tense: "Add feature" instead of "Added feature."
- Keep it brief but descriptive. A good message is less than 72 characters long and describes the change succinctly.

Thank you for contributing to Eaglercraft-1.21.11!
